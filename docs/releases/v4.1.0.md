# v4.1.0

发布日期：2026-04-19

## 升级摘要

- 统一了 `Quant Lab` 与 `Research Workbench` 的一级页面骨架，首屏新增 hero、共享上下文栏和 workspace surface，研究工作区的第一屏不再只是组件堆叠。
- 修正了前端深链入口与后端落页语义，`/quantlab`、`/pricing` 等路径别名现在会稳定归一到 `?view=` 视图 URL，刷新和分享链接更可靠。
- 告警编排的 `persist_event_record=false` 现真正表示“不持久化事件记录”，不会再无条件落历史和事件表。
- 后端 UTC 时间调用完成清理，避免继续依赖 `datetime.utcnow()`，同时保持现有无时区后缀的 UTC 存储格式不变。

## 重点升级

### 1. 页面骨架与首屏层级

- `Quant Lab` 现在具备统一 hero、工作区摘要、焦点说明和快捷切换入口，实验与运营面板在同一个一级页面结构下展开。
- `Research Workbench` 新增共享视图 context rail、首屏指标摘要与右侧焦点卡，批量推进、批量评论和当前任务操作都前置到了更稳定的位置。
- 两个页面都补了响应式收口，移动端会回落成单列布局，避免首屏横向撑开。

### 2. 深链与研究上下文

- 视图路由开始统一收敛到 `?view=`，从外部直接打开 `/quantlab`、`/pricing` 这类入口时，不会再误回默认视图。
- 工作台复制链接和从研究页回跳时能更稳定地保留筛选条件、当前任务和队列上下文。

### 3. 告警编排与后端语义

- `persist_event_record=false` 不再只是前端表单选项，而是真正影响后端写入行为。
- 当该开关关闭时，告警编排历史、`alert_event` 与 `alert_event_dispatch` 都不会被写入，运营和审计视图不会再被无意污染。

### 4. 时间处理与工程卫生

- 后端的 UTC 时间戳统一切到 `datetime.now(timezone.utc)` 体系，减少弃用 API 带来的警告和潜在时间语义偏差。
- 发版链路继续沿用 `VERSION` 作为单一版本源，同步更新前端元数据和 API 文档。

## 验证情况

- 前端壳层与工作台相关测试通过：`quant-lab-shell`、`workbench-shell`、`research-workbench-copy-link`、`workbench-detail-panel`、`workbench-detail-sections`
- 后端相关单元测试通过：`test_quant_lab_alerts.py`、`test_realtime_journal.py`、`test_realtime_alerts.py`、`test_research_workbench_endpoint.py`
- 真实浏览器补充验证了 `Quant Lab` 与 `Research Workbench` 的桌面/移动端首屏，无新增页面运行时异常，且未出现横向溢出

## 升级提示

- 如果你们有直接引用 `/quantlab`、`/pricing` 等路径的书签或内部链接，升级后会自动规范到 `?view=` 形式，这是预期行为。
- 如果运营流程此前依赖 `persist_event_record=false` 但仍能在历史里看到事件，升级到 `v4.1.0` 后该行为会被修正，建议同步确认相关报表或巡检逻辑。
- 若本地或测试环境对 UTC 时间字符串格式有自定义解析，升级后无需改格式契约，但建议重新跑一遍依赖时间排序或快照时间线的流程。
