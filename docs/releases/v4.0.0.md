# v4.0.0

发布日期：2026-04-14

## 升级摘要

- 基础设施层从“可选辅助能力”升级为正式产品能力，新增 `Infrastructure` API、认证令牌、持久化状态、Redis/Celery 任务队列、通知能力与 TimescaleDB schema，并提供一键启动、迁移与健康检查脚本。
- Quant Lab 上线为独立量化实验台，统一承载策略优化、批量回测、组合实验、Walk-Forward、风险中心、交易日志、告警编排与估值实验。
- GodEye 从宏观错误定价扩展到结构性衰败与部门混乱监控，新增 people / governance / execution / physical / evidence 五层雷达，以及部门执行混乱、物理世界跟踪和交易论点观察面板。
- Research Workbench、实时复盘、行业研究、跨市场分析与定价研究继续模块化，并将更多偏好、提醒、快照和上下文切换能力接入持久化与异步处理链路。

## 重点升级

### 1. 基础设施与运维

- 新增 `/infrastructure/*` 端点，覆盖登录、token 刷新、OAuth provider 配置、任务队列状态、通知配置、限流状态与配置版本查询。
- 新增 `docker-compose.quant-infra.yml`、`start_infra_stack.sh`、`start_celery_worker.sh`、`migrate_infra_store.py` 等脚本，支持单独拉起 PostgreSQL/TimescaleDB、Redis 与 worker。
- 新增 TimescaleDB schema 与持久化支撑层，为研究任务、通知与速率控制状态预留正式存储能力。

### 2. Quant Lab

- 新增 `Quant Lab` API 与前端工作区，支持策略优化、批量实验、基准比较、组合构建、Walk-Forward 验证和风险中心分析。
- 交易日志、提醒编排、数据质量检查和估值实验统一纳入量化实验台，减少在多页面之间来回切换。
- 高级回测实验台进一步拆分为独立 section/hook，后续扩展实验模板与结果解释会更稳定。

### 3. GodEye 与宏观研究

- 新增结构性衰败雷达，将 people、governance、execution、physical、evidence 五个层面的退化信号统一收口成可视化监控面板。
- 新增部门混乱看板、people layer watchlist、physical world tracker、trade thesis watch 等面板，把宏观叙事与执行证据放到同一研究界面。
- 宏观与政策信号现在更直接参与跨市场与研究工作台的上下文切换，便于从观察走到行动。

### 4. 研究工作流

- Research Workbench 继续细分为 overview、detail、snapshot compare、selected task intelligence 等模块，复制链接、优先级事件、上下文版本对比能力更完整。
- 实时行情、行业研究、定价研究与跨市场分析补齐更多共享状态与异步动作，使偏好、提醒、快照与派生解释更容易持久化。
- 配套测试继续扩展，前后端围绕 Quant Lab、GodEye 新面板、Workbench 新交互和基础设施链路补齐更多回归覆盖。

## 升级提示

- 建议前后端与基础设施脚本同时升级到 `v4.0.0`，尤其是在启用持久化、任务队列或通知能力时，避免接口与存储结构不一致。
- 首次使用基础设施模式时，建议运行 `./scripts/start_infra_stack.sh --bootstrap-persistence` 并执行 `python3 ./scripts/migrate_infra_store.py` 完成初始化。
- 如果已有本地研究偏好或快照缓存，升级后建议重新验证关键工作流，确认新的持久化与异步行为符合预期。
