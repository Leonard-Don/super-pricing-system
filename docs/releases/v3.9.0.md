# v3.9.0

发布日期：2026-04-02

## 升级摘要

- 宏观错误定价引擎升级为 6 因子可靠度体系，新增利率曲线压力、信用利差压力、汇率错配，并为证据冲突、覆盖度、时滞、来源漂移、反转前兆和政策源健康度提供统一解释。
- 定价研究完成模块化重构，资产定价、估值、Gap Analysis、标的检索和同行对比支撑层解耦，CAPM / FF / DCF 结果展示与扩展能力更稳定。
- 实时行情与研究工作台继续结构化，复盘快照/时间线进入版本化管理，分享模板、诊断开关与数据派生逻辑统一收口。
- 行业研究配置开始支持后端同步、导入导出和流式构建状态反馈，自选行业、保存视图与提醒阈值可随版本一起沉淀。

## 重点升级

### 1. 宏观研究

- 新增 `rate_curve_pressure`、`credit_spread_stress`、`fx_mismatch` 三个宏观因子，并接入统一注册表。
- 宏观证据摘要新增 conflict summary / trend、coverage、cross confirmation、source dominance、source drift、source gap、reversal precursor 等质量维度。
- 政策源健康度开始直接影响输入可靠度与置信度惩罚/加成逻辑。

### 2. 定价研究

- 定价研究后端拆分为独立支撑模块，覆盖标的建议、同行候选池、基准因子摘要、敏感性参数与估值辅助逻辑。
- 前端拆分为搜索、概览、模型卡片、结果区与洞察卡片等独立模块，降低大组件维护成本。
- 资产定价、估值与 Gap Analysis 的解释层复用统一支撑函数，结果一致性更好。

### 3. 实时 / 行业 / 工作台

- 实时行情面板拆出派生状态、元数据与分享模板，复盘快照和时间线加入版本号兼容处理。
- 行业页支持导出/导入行业偏好配置，并将自选、保存视图和提醒阈值回写后端。
- 行业成分股构建支持流式状态反馈；GodEye 与 Research Workbench 继续拆分为更清晰的子模块。

## 升级提示

- 建议前后端同时升级到 `v3.9.0`，避免前端新结构与旧接口字段不一致。
- 首次打开行业页时会合并浏览器本地偏好与后端偏好数据。
- 若需要打开实时诊断，可在地址栏使用 `?realtimeDiagnostics=1`；关闭可使用 `?realtimeDiagnostics=0`。
