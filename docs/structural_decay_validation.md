# Structural-Decay Signal Validation

Reproducible test: `python3 scripts/validate_structural_decay.py`. Generated by that script -- do not edit by hand.

## 1. What was tested

The portfolio review flagged that `src/analytics/structural_decay.py` and `src/analytics/macro_mispricing_thesis.py` emit a narrative + trade legs with **zero statistical validation and no backtest** (`grep mispric src/backtest/` returns nothing).

This validates ONE concrete, falsifiable sub-claim of that machinery:

> **A worse (higher) structural-decay score predicts a lower subsequent N-month excess return.**

## 2. Method

- **Universe:** the 11 names that have a curated people-layer record AND usable price history: `0700.HK, AAPL, AMZN, BABA, BIDU, GOOGL, META, MSFT, NVDA, TSLA, TSM`.
- **Benchmark for excess return:** `SPY` (the engine's own fallback market-hedge leg in `macro_mispricing_thesis.py`).
- **Anchors:** monthly (~21 trading days apart) over `5y` of real daily prices from yfinance. 40 anchor dates, 2022-05-09 -> 2025-11-05.
- **Score, point-in-time:** at each anchor `t` the structural-decay components are reconstructed using ONLY data up to `t`:
  - `execution_decay`: trailing 252-day CAPM alpha (replicating `AssetPricingEngine._run_capm`: daily OLS, annualised x252). **Genuinely walk-forward.**
  - `people` components: from the curated `people_layer.json` snapshot. Verbatim delta thresholds from `build_structural_decay`.
- **Forward return:** stock return over the next N months MINUS the `SPY` return over the same window -- strictly post-anchor data. Horizons N = 1m, 3m, 6m.
- **Statistics:** per-anchor Spearman rank IC; mean IC with a t-stat across anchors; share of anchors with IC<0 (hit rate); a 5000-sample bootstrap 95% CI on the mean IC; and a median-split long/short forward-return spread with its own t-stat.

## 3. Honest data-quality caveats

1. **No historical score panel exists.** `build_structural_decay` only ever runs point-in-time inside a request; nothing persists a time series of scores. This script *reconstructs* the panel -- it is not reading a stored one.
2. **The people-layer file is a CURATED STATIC SNAPSHOT.** Every snapshot date in `people_layer.json` carries an identical `people_fragility_score` per name (`source_mode = curated`, `fallback_reason = live_proxy_or_def14a_not_connected`). It therefore contributes only a fixed per-name offset and cannot react to anything. The execution-only test (A) is the clean read.
3. **FF3 alpha is proxied by the CAPM alpha.** There is no offline SMB/HML factor panel, so the FF3 branch of `execution_component` is approximated. This only nudges the -3/-5 threshold gating.
4. **Risk-free rate is treated as ~0** over each window (daily FF RF is ~1e-4 and does not move a cross-sectional ranking).
5. **Small universe (11 names).** A cross-section this narrow makes any single-anchor IC noisy; this is why the test aggregates IC across many anchors and bootstraps the mean.

## 5. Verdict

NOT VALIDATED -- across every tested horizon the structural-decay signal showed no rank IC distinguishable from zero. On the available data the mispricing thesis has no measurable predictive power.

Read this the same way as the quant-trading-system walk-forward stat tests: a null result, honestly reported, is the deliverable. If the table above shows IC t-stats inside +/-2 and bootstrap CIs straddling 0, then **the structural-decay signal does not currently predict forward returns on the data this repo can assemble** -- and the narrative/trade-leg output of `macro_mispricing_thesis.py` should be treated as an un-evidenced hypothesis, not a validated signal.

## 6. The #1 thing to fix

To make the mispricing thesis testable for real, the repo needs a **persisted historical panel of structural-decay scores** (and of the people-layer inputs that feed them) captured point-in-time, not a curated static snapshot. Until a genuine longitudinal panel exists, every backtest of this signal is a partial reconstruction with the caveats listed in section 3. Persisting the scores -- one append per analysis run, keyed by `(symbol, timestamp)` -- is a small change and is the single highest-leverage fix for the validation gap.
